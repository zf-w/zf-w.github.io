Concepts learned, reinforced, and practiced

I have learned, reinforced, and practiced a lot of interesting concepts while working on this project.

General Tech stack

  • Core: Rust Programming Language:

This programming language and its core conceptual design really help me to think and design more carefully and, as a result, quicker and safer. The Rust Programming Language also has a well-designed project structure and tutorial readings.

  • Backend: Rust-Actix-Web

A Rust web library that enables me to fairly easily connect my Connect Four solving codes onto it with its design that allows passing shared information between threads.

  • Backend: Heroku

Heroku is a platform as a service that enables developers to build, run, and operate applications in the cloud. I find its workflow very convenient. I learned how to make my own build pack and deployed the whole thing with the push of a button.

  • Frontend: Next.js

This tool provided me a fairly fast way to build component-based web pages. But I'm hoping to find other, more lightweight alternatives.

  • Management: Git+GitHub

Very useful version control system. Most importantly, they provide me a way to record my everyday work in a diary.

Development

  • Cargo Workspace and Git Submodules

With these two, I can easily break down large projects into separate components that can be shared in another project of mine. Again, I'm not sure what the best practice is. But I feel making some core codes into separated repositories and cloning them into other projects are indeed natural and convenient.

  • Heroku build pack and Shell

Customize your deployment process and caching between builds.
