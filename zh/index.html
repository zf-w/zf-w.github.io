想法与实践们

这本"书"

这本书的实现是我很长一段时间的怨念了. 很长时间以来, 我都在追寻一种可以积累, 透明, 技术上灵活又便捷的形式去记录自己的各种东西. 总的来说或许是吃饱了撑的. 但是我确实希望能有些不一样的.

我希望这个东西能漂亮而我不用常常在意排版, 字号, 从而尽可能不受工具所限(INFO303, 2022). 我希望这个东西能既灵活的能够加入我的各种想法, 比如说你看到嘟上面意义不明的网格, 一方面我又能尽可能自己掌握而不受比如说别人的控制, 我可以做成网页, 也可以打印, 也可以刻在个碑上之类. 当然, 虽然这样的话, 上面的东西就不能转了, 不过好在反正就是 2D 的. 另一方面, 虽然技术上需要有很多设计, 但是又足够灵活而不至于被限制在一种技术框架内. 同时, 我希望写作的过程也足够人性化, 不至于让我老思考机器该如何读取, 总之不能有太多尖括号或者花括号之类.

游戏研究

游戏, 某种角度上说是一种对世界的抽象. 相比世界, 或许游戏能提供一些独特的角度让我去理解现实世界中的一些问题.

上面这个图的游戏叫做 Connect Four. 我也不知道该咋翻译, 就叫竖四子棋吧. 和四子棋不一样, 它只能从上往下堆叠落子. 图中棋子的数代表了这个从 0 开始的第几步. 图里的局面理论上来说是后手的最优的局面呢. 他如果走错的话, 就会导致输的更快的. 多么残忍! 小小的游戏里似乎就能蕴含无限的痛苦和挣扎呢. 或许能对现实也提供一些思考的巴. 这里是去 Demo 的 链接. 然后这里是去随笔的 链接

网络的可视化

对网络, 或者说图的可视化也是我思考以及实践很长时间的一个项目. 我从 2019 年就开始逐步实践, 学习. 一开始的初衷是希望能可视化一个游戏然后用它对我玩这个游戏过程中不太礼貌的行为道歉. 到去年也算是兑现了我对自己的承诺. 从阅读文献, 到在 Python 里实现, 到设计如何读取前人的数据设计我自己的储存方式, 到学习网页 3D, 3D 渲染的思路, 学习了校内外的课程后,再用 C++和 Rust 更加规整, 系统地实现 到最终做成成品, 和人道歉...真的是好长好长好长的一段路呢. 我似乎一辈子也再干不了几次这样的事情了. 当然最好也不要再有类似的尴尬以至于需要和人道歉.

其它各种源源不断的想法

上面说的是几个跨度非常大, 又相对完整, 全面的几个小项目. 在思考上面的东西的过程中, 也产生了很多别的小创意. 比如说可以用 3D 打印的形式把很漂亮的分形图案打印出来, 比如说既然 3D 模型的本质都是网络, 那可不可以在 3D 模型上运行一些算法程序, 好比说服装上. 类似的小点子还有很多, 有机会慢慢说.
